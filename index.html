Unit 0: Basics of the Web

World Wide Web

The web is a collection of HTML documents connected by hyperlinks. There are about 30 billion pages. Browser requests via the internet to servers use HTTP. Servers respond with files that the browser displays, and servers host the files.

HTML

HTML, or hypertext markup language, provides the structure. It contains text content – what you see – and markup – what it looks like. You can also enter comments (HTML: !comment; CSS: / * comment * /; Python: #).

Tags and Elements

Tags are formatted with the opening tag, "content", closing tag. Both tags and content form an element. If a tag has no end tag, it is a void tag. There are inline, block, and container tags. Elements like h1, h2, etc. contain semantic meaning and style rules.

White Space

All spaces (tabs, enters, multiple spaces) are viewed as a single space in HTML. You can use br (break) tag or p (paragraph) tag to add in extra spaces.

HTML Documents

HTML documents contain: doctype, HTML tag, head tag (CSS styling goes here), title tag (title of page in top of browser window/tab), body tag (content of document).

Unit 1: Structuring a Document

Developer Tools

You can access HTML and CSS of a webpage via the Developer Tools; however, not all elements are in structure (ie header). You can click the element in the tools and see details (font size, border width, color). Conversely, you can also right-click the actual element and then "Inspect Element" and see details in Developer Tools. All elements are rectangular, as boxes make it easier to arrange elements. You use div to define boxes.

DOM

Elements are contained via the tree-like/branching structure called Document Object Model (DOM). This is how the browser organizes tags, elements, content.

CSS

CSS provides the style; pages can have identical HTML but look totally different because of CSS. You can give "class" attribute to each element with unique names in order to tell what they are.

Unit 2: Using CSS to Add Style

Adding CSS

You need to add certain tags (!DOCTYPE html and html) to tell browser how to interpret code. You add head tag and then either put in style that will apply to the full document or put the link to the file of style code. In this file, you'll define class titles with the style that you want. You can also add style in the HTML file by using the style tag, but this leads to redundancies if it is done inline.

Cascading Style Sheets

CSS stands for Cascading Style Sheets and the "cascading" part means that the most specific rule is applied to an element. Inheritance is the idea that properties are applied to the descendants of an element; properties like color, font, etc. are inherited from ancestor elements. This makes it so you don't have to re-declare properties, eliminating redundancies. Using inheritance to apply styles to multiple elements without having to rewrite them allows developers to write less CSS, which speeds up loading of web pages and saves money on development costs.

Selectors

Selectors define the elements that styles will apply to. Style rules will apply to all tags that you define; you can use established element types (like h1 or p) to define all of those types of elements or use class selectors to name elements to style them differently based on class names. The CSS reference list is very helpful in listing and explaining the CSS attributes.

The Box Model

The Box Model contains the components of an element: margin (no background color, the space between boxes), border (surrounds padding/content), padding (space in box around content), and content (image, text). You can calculate the size of an element by adding together the pixels of all of these components. If you set the size as a percentage, the element will adjust depending on the screen size. You can also position elements by using { display: flex; }, which will allow divs to appear next to each other.

Testing CSS

When making your document, first start with structure. Then move on to defining and using tags, sizing elements, and finally positioning them. Then you can test and refine to make small fixes (font, color, etc). You can test the validity of your CSS and HTML codes to make sure that it will work for all browsers.

Unit 2: Intro to "Serious" Programming

Computers & Programs

Computers are defined as being universal machines that can do any computation that you can write a program for, and they can execute billions of instructions very quickly. Programs are precise sequence of steps necessary to tell a computer what to do and make it useful. High-level languages like Python avoid the ambiguity and verbosity that a natural language would bring and use interpreters. Input is the code given to an interpreter, which interprets and executes the code.

Python

Precise use of spacing and grammar are important in Python. The Backus-Naur form is used for expressions, which uses derivation to replace non-terminals with other non-terminals, a combination of non-terminals, and finally terminals. Arithmetic grammar, for example, uses Expression-Operator-Expression format; as a recursive definition, it needs terminal and non-terminal rules to allow derivation. Also, Python will put an answer in integers if all numbers in the expression are integers.

Unit 3: Variables & Strings

Variables

Assigning a name (variable) to an value (expression) makes code readable and meaningful. Assignment statements link the variable name to the value, and they need to start with a letter or an underscore (convention is to use lowercase). You can then use the variable in calculations, and variable values will be updated if calculations change. In this way, you can use the same variable in the assignment statement because the right side of the assignment statement is evaluated first.
Example: if days=48, then output of days=days-1 is 47.

"=" does not mean the statement is an equality; it means the assignment of the right side's value to the variable on the left.

Strings

Strings are a sequence of characters surrounded by quotes (single or double; must match). "+" can be used to concatenate strings, but you'll need to manually add spaces. You can also multiply strings to repeat the strings. You can't concatenate strings with integer objects.

Indexing Strings

Indexing strings is extracting sub-sequences from a string. All extractions use square brackets, and the first character in a string starts at 0. Using negative numbers counts from the end of the string. You can do one character extraction or subsequence extraction. In subsequence extraction, you can select a range or everything up to or after a position. In one character extraction, if there is no character in a certain position, you'll get an error. In contrast, if there are no characters in a subsequence range, you'll get a blank output.

Find

Find is a method/procedure that reports the first occurrence of a target string in a search string. If the target string is not in the search string, the output is -1. You can add parameters to say at what position to start the search in order to find later occurrences.

Unit 4: Functions

Functions
(Procedures)

Functions map inputs to outputs. Use "def" to define your own functions (some, like "find", are built in) and parameters. Make sure to put a colon after the end of the parameters! You can combine functions or use them multiple times in print statement. This powerful tool is called procedure composition. Once you define a function, it can be used forever and thus is hugely important in allowing programmers to avoid repetition in programming, therefore saving time and money. Defining functions and calling them up is the foundation of computer programming.

Input & Output

Input is information that is passed into the parameters of the function. Output is the end result, what is printed after the function is executed (also known as "calling a function"). There needs to be a return statement within a function to see the output. The values of input variables do not change outside of the function, so to see the updated values you need a return statement.

Making Decisions

If and Else

You can use an If Statement for comparison to generate different outcomes. Additionally, if the test expression in the if statement is false, you can use "else" to set up action to take. != means not equal while == means equality (not assignment).

Or Operator

Or can be used to evaluate different expressions but things will only be evaluated if needed; if first expression is true, the second expression is not evaluated. If first expression is false, then the outcome is the value of the second expression.

While Loops

While loops continue to loop back and run as long as the test expression is true. A loop that doesn't have a test expression is an infinite loop, or a bug, and the computer will get stuck. "Break" ends the loop and moves forward even if the test expression is still true.

Troubleshooting

When troubleshooting code, first look at the traceback (error code that can direct you to location and description of error). You can also use example code, check that it works, and then adapt to your needs. Test step by step (and test intermediate variables). Also, comment out different versions of code so you can compare what does and doesn't work.

Structured Data

Lists

Lists are a sequence of anything; they don't have to be comprised of just characters or just numbers. You can even have nested lists where an expression in a list is another list. Use square brackets and separate the expressions by commas. A list of [] is an empty list. You can index a list and its expressions the same way that you do a string. If there is a nested list, you can index the expression (the list) and then the target item within the nested list.

Mutation & Aliasing

Mutation is the ability to change the original value of something by replacing it, and lists support mutation. Strings and numbers are not mutable; they create a new string and disassociate with the original. This makes it important to pay attention if multiple variables reference an object that you change. Aliasing references referring to one object in different ways. Again, if you change the object, it affects everything that references it. You can also append [list.append(element)], which is adding element to end of list and is mutation. You can concatenate lists with "+" and this produces a new list via assignment, not mutation. Length [len(list)] coubts the number of elements (outer elements only; not components of list-within-list) in a list or number of characters in a string.

For Loops & Indexing in Lists

For loops are a simpler way to do while loops for lists. "For name in list:" applies the block expression to each element (which is assigned the new name) until the end of the list. Indexing returns the first place in a list where the value occurs: list.index(value). It will return an error if the value isn't in the list. You can check if the value is in the list (T/F) with "print value in list".

Unit 4: Object-Oriented Programming

Object-Oriented Programming

Object oriented programming is a different way of structuring programming that is focused on objects (instead of the logic used to create them).

Classes

Classes are incredibly important. They allow you to apply a template of code to create multiple instances or objects. This is incredibly valuable for large programming projects that have repeated and similar objects. Using object oriented programming lets you create a single template (the class) to create all of these instances, which saves a lot of time, effort, and money. Using classes also make code modular, or separates functionality into modules that can be used again in other locations. A class can be seen as a "blueprint" that allows you to create different instances with different specifications. There is an "init" function within the class that creates space and memory for a new instance of that class. The init function is written as "__init__" and the underscores indicate that it is a reserved function of Python. When you create a new object or instance, you use "self" as the argument of the function definition.

Class & Instance Vocabulary

  • Class: blueprint, contains data and methods
  • Instance: a specific version of a class; one output of a class using particular specifications
  • Object: the output of object oriented programming; each instance of a class is an object
  • Constructor: the function (init) used to create the instances; uses "self" as argument
  • Self: instance being created bt the constructor
  • Instance variables: variables unique to the instance of a class; self.____
  • Instance methods: a function defined within a class and associated with an instance
  • Class variables: associated with a class; relevant to all instances and written outside of init definition (pre-existing class variables in python include __doc__, __name__, and __module__). A module is a file with definitions that you can reference and import into your working file.

Inheritance

Inheritance also allows you to further reuse code (by importing it from a previous class and adding to it or tweaking it for the purposes of your current class), which is a beneficial aspect of object-oriented programming and saves time and money for programmers. It also allows your code to more intuitively match how we think. It references previously defined classes and you can use methods and variables from there. If you have a method in the newest class that has the same name as a method in the previous class that you are referencing, the new method will override the old (method overriding). This is similar to CSS's cascading structure, which applies the most specific rule to an element but otherwise using inherited rules. Both allow you to make the most of previously written code while tweaking any minor differences.

Resources

Python Standard Library contains modules and their functions. Stackoverflow is also a good forum resource. Google Style Guide for Python also has good conventions to follow (e.g. label constant variables in all caps or write class definitions in one file and the code that calls that class in order to create instances in another file).
